=============== 구현한 내용, 구현하지 못한 내용 ===============

숙제에서 요구된 모든 사항에 대해서 구현이 완료되었습니다.
다만 숙제에서 의도한 바와 세세하게 다를 수 있는 부분이 있어 해당 사항만을 적어두었습니다.

o/p: 투영을 선택한다 (직각 투영 / 원근 투영)
- 직각 투영은 1/3인칭에서 적용하면 매우 어색해서, 맵 전체를 바라봤을 때의 시점으로만 적용됩니다

z/Z: 원근 투영 시 z축으로 이동할 수 있게 한다.
- 버튼을 누를 시마다 조금씩 z축으로 이동합니다. 카메라가 바라보는 AT은 항상 (0, 0, 0)으로 고정해두었습니다.
  z축 이동은 수치 상으로는 문제가 없으나 카메라 회전 후 사용할 경우,
  z축 방향으로 일직선의 레일을 따라 카메라가 항상 (0,0,0)을 바라보는 효과가 생겼습니다.

y/Y: 카메라가 바닥의 y축을 기준으로 양/음 방향으로 회전한다.
- 바닥의 y축이라는게 월드 좌표 기준이라고 생각하여 월드 y좌표에 대해 공전합니다.
  1/3인칭은 적용되지 않습니다!!

r: 미로를 제작한다.
v: 육면체들 움직임이 멈추고 낮은 높이로 변한다.
s: 미로에서 객체가 나타난다.
- 해당 사항들은 애니메이션 없이 즉시 변환으로 구현했습니다.

→/←/↑/↓: 객체를 미로에서 앞/뒤/좌/우 이동. 미니맵에서 객체 이동 확인할 수 있어야 함.
- xz 평면에서 전후좌우 이동하는데, 플레이어에 회전 기능이 없기 때문에 x축과 z축 +- 방향으로만 움직입니다.

1/3: 카메라 시점 1인칭/3인칭 변환
- 미로의 가로세로 길이가 고정된 상태에서 입력값만큼 벽 개수를 나누는데, 때문에 벽 개수가 일정량 이상 많아질 경우
  카메라 시야각에 의해 플레이어 양옆의 벽 안을 뚫고 본다거나, 전방의 벽 너머를 투시해서 볼 수 있는 문제가 있습니다.

c: 모든 값 초기화
- 가로세로 개수를 입력 받을때로 넘어갑니다.

q: 프로그램 종료



=============== 키보드/마우스 명령어 ===============

숙제 키와 동일합니다
v, s, 1, 3은 토글이 가능한 키입니다.


=============== 추가 구현 요소 ===============

1. 빛을 적용시켰습니다.
다만 다른 오브젝트에 가려 그림자가 지진 않습니다.

2. 미로 생성이 랜덤하게 생성됩니다.
구현 방식을 고민하다 미로 생성 알고리즘을 검색해봤고, Prim 알고리즘을 통해 만드는 방식을 선택해 구현했습니다.
다만 알고리즘을 처음부터 직접 만들진 않고, https://choi-dan-di.github.io/algorithm/prim-algorithm/ 해당 사이트 코드를 참고하였고
원인 불명 버그는 코파일럿 적극 활용하여 디버깅 했습니다
